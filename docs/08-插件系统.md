# 插件系统

插件系统兼容 claude-code plugins 格式，支持通过插件扩展命令、Hook、Agent、MCP 等能力。

> 对应源码：`src/openharness/plugins/`

---

## 模块组成

| 文件 | 职责 |
|------|------|
| `loader.py` | 插件发现、加载、解析 |
| `installer.py` | 插件安装 |
| `schemas.py` | `PluginManifest` — 插件清单 Pydantic 模型 |
| `types.py` | `LoadedPlugin` — 加载后的插件数据 |

## 插件目录

插件从两个位置加载：

```python
def get_user_plugins_dir() -> Path:
    return get_config_dir() / "plugins"          # ~/.openharness/plugins/

def get_project_plugins_dir(cwd) -> Path:
    return Path(cwd) / ".openharness" / "plugins"  # 项目级 .openharness/plugins/
```

## PluginManifest 清单

> 源码：[`plugins/schemas.py`](../src/openharness/plugins/schemas.py)

```python
class PluginManifest(BaseModel):
    name: str                              # 插件名称
    version: str = "0.0.0"                 # 版本
    description: str = ""                  # 描述
    enabled_by_default: bool = True        # 默认是否启用
    skills_dir: str = "skills"             # 技能目录名
    hooks_file: str = "hooks.json"         # Hook 文件名
    mcp_file: str = ".mcp.json"            # MCP 配置文件名
```

## 插件发现流程

> 源码：[`plugins/loader.py`](../src/openharness/plugins/loader.py)

```python
def discover_plugin_paths(cwd) -> list[Path]:
    roots = [get_user_plugins_dir(), get_project_plugins_dir(cwd)]
    paths = []
    for root in roots:
        for path in sorted(root.iterdir()):
            if path.is_dir() and _find_manifest(path) is not None:
                paths.append(path)
    return paths
```

Manifest 查找支持两种位置（兼容 claude-code）：

```python
def _find_manifest(plugin_dir: Path) -> Path | None:
    for candidate in [
        plugin_dir / "plugin.json",              # 标准位置
        plugin_dir / ".claude-plugin" / "plugin.json",  # claude-code 兼容
    ]:
        if candidate.exists():
            return candidate
    return None
```

## 插件加载流程

> 源码：`loader.py:load_plugin()`

```python
def load_plugin(path, enabled_plugins) -> LoadedPlugin | None:
    # 1. 查找并解析 manifest
    manifest_path = _find_manifest(path)
    manifest = PluginManifest.model_validate_json(manifest_path.read_text())

    # 2. 判断是否启用
    enabled = enabled_plugins.get(manifest.name, manifest.enabled_by_default)

    # 3. 加载技能 (skills/, commands/, agents/ 目录)
    skills = _load_plugin_skills(path / manifest.skills_dir)
    skills.extend(_load_plugin_skills(path / "commands"))
    skills.extend(_load_plugin_skills(path / "agents"))

    # 4. 加载 Hooks
    hooks = _load_plugin_hooks(path / manifest.hooks_file)
    if not hooks:
        hooks = _load_plugin_hooks_structured(path / "hooks" / "hooks.json", path)

    # 5. 加载 MCP 服务配置
    mcp = _load_plugin_mcp(path / manifest.mcp_file)
    if not mcp:
        mcp = _load_plugin_mcp(path / ".mcp.json")

    return LoadedPlugin(
        manifest=manifest,
        path=path,
        enabled=enabled,
        skills=skills,
        hooks=hooks,
        mcp_servers=mcp,
        commands=[s for s in skills if s.source == "plugin"],
    )
```

### 智能组件发现

插件可以在多个目录下放置不同类型的组件：

| 目录 | 加载为 | 说明 |
|------|--------|------|
| `skills/` | SkillDefinition | 领域知识 |
| `commands/` | SkillDefinition (source="plugin") | 命令（Markdown 格式） |
| `agents/` | SkillDefinition | Agent 定义 |
| `hooks/hooks.json` | Hook definitions | 生命周期钩子 |
| `.mcp.json` | MCP server config | MCP 服务 |

## Hook 加载的两种格式

### 格式 1：简单 JSON

```json
{
  "pre_tool_use": [
    {"type": "command", "command": "echo check", "matcher": "bash"}
  ]
}
```

### 格式 2：结构化 JSON（claude-code 兼容）

```json
{
  "hooks": {
    "pre_tool_use": [
      {
        "matcher": "edit_file",
        "hooks": [
          {
            "type": "command",
            "command": "${CLAUDE_PLUGIN_ROOT}/scripts/check.sh $ARGUMENTS"
          }
        ]
      }
    ]
  }
}
```

`${CLAUDE_PLUGIN_ROOT}` 变量在加载时被替换为插件实际路径：

```python
cmd = hook.get("command", "")
cmd = cmd.replace("${CLAUDE_PLUGIN_ROOT}", str(plugin_root))
```

## LoadedPlugin 数据结构

```python
@dataclass
class LoadedPlugin:
    manifest: PluginManifest            # 插件清单
    path: Path                          # 插件目录路径
    enabled: bool                       # 是否启用
    skills: list[SkillDefinition]       # 技能列表
    hooks: dict[str, list]              # Hook 定义
    mcp_servers: dict[str, object]      # MCP 服务配置
    commands: list[SkillDefinition]     # 命令列表
```

## 插件管理 CLI

```bash
# 列出已安装插件
oh plugin list

# 安装插件
oh plugin install /path/to/plugin
oh plugin install https://github.com/user/plugin-repo

# 卸载插件
oh plugin uninstall my-plugin
```

在交互会话中：
```
/plugin list       # 查看插件
/plugin enable X   # 启用插件
/plugin disable X  # 禁用插件
/reload-plugins    # 重新发现插件
```

## 插件与其他系统的集成

```
load_plugins()
    ├── skills → 注入到 SkillRegistry      → system prompt 中展示
    ├── hooks  → 注入到 HookRegistry       → 工具执行时触发
    ├── mcp    → 注入到 MCP 服务配置       → MCP 工具可用
    └── commands → 注册为可用命令           → / 命令列表中展示
```
