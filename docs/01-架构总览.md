# OpenHarness 架构总览

## 项目定位

OpenHarness 是一个用 Python 编写的轻量级 **AI Agent Harness（代理装具）** 框架。它围绕 LLM 提供完整的 Agent 基础设施：工具调用、技能加载、权限管理、多智能体协调等。

> **核心理念**: 模型提供智能，Harness 提供双手、眼睛、记忆和安全边界。

与 Claude Code 对比：代码量仅 ~11,700 行（对比 512,664 行），44 倍精简，但覆盖 98% 的工具能力。

## 10 大核心子系统

| # | 子系统 | 模块路径 | 核心职责 |
|---|--------|----------|----------|
| 1 | **Agent Loop** | `engine/` | 核心对话循环：流式请求 → 工具调用 → 结果拼接 → 循环 |
| 2 | **API 客户端** | `api/` | Anthropic 兼容 API 调用，指数退避重试 |
| 3 | **工具系统** | `tools/` | 43 个工具的注册、Schema 生成、执行 |
| 4 | **权限系统** | `permissions/` | 三级模式 + 路径规则 + 命令黑名单 |
| 5 | **Hooks** | `hooks/` | PreToolUse/PostToolUse 生命周期事件 |
| 6 | **技能系统** | `skills/` | 按需 Markdown 技能加载 |
| 7 | **插件系统** | `plugins/` | claude-code 兼容的插件生态 |
| 8 | **记忆 & 上下文** | `memory/` + `prompts/` | MEMORY.md + CLAUDE.md + System Prompt 组装 |
| 9 | **多智能体** | `coordinator/` + `tasks/` | 后台任务管理 + 团队协调 |
| 10 | **命令 & UI** | `commands/` + `cli.py` + `ui/` | 54 个命令 + Typer CLI + React TUI |

## 核心数据流

```
                           ┌────────────────────────────────────────┐
                           │               用户输入                 │
                           └───────────────┬────────────────────────┘
                                           ▼
                               ┌───────────────────────┐
                               │    cli.py (Typer)      │
                               │  解析参数，配置加载    │
                               └───────────┬───────────┘
                                           ▼
                            ┌──────────────────────────────┐
                            │      RuntimeBundle 组装      │
                            │  API Client + ToolRegistry   │
                            │  PermissionChecker + Hooks   │
                            │  Skills + Plugins + Memory   │
                            └──────────────┬───────────────┘
                                           ▼
                         ┌─────────────────────────────────────┐
                         │         QueryEngine                 │
                         │  维护 messages[], cost_tracker      │
                         │  submit_message(prompt) → 事件流    │
                         └──────────────┬──────────────────────┘
                                        ▼
              ┌────────────────────────────────────────────────────┐
              │             run_query() 核心循环                   │
              │                                                    │
              │  for turn in range(max_turns):                     │
              │    ┌──► stream_message(messages, tools) ◄─────┐   │
              │    │    ├─ yield TextDelta (流式文本)           │   │
              │    │    └─ yield MessageComplete (含 tool_uses)│   │
              │    │                                           │   │
              │    │    if no tool_uses: break                  │   │
              │    │                                           │   │
              │    │    for each tool_call:                     │   │
              │    │      1. PreToolUse Hook 检查               │   │
              │    │      2. ToolRegistry.get(name) 查找工具    │   │
              │    │      3. Pydantic input_model 验证输入      │   │
              │    │      4. PermissionChecker.evaluate() 权限  │   │
              │    │      5. tool.execute() 执行                │   │
              │    │      6. PostToolUse Hook 通知              │   │
              │    │                                           │   │
              │    └── messages.append(tool_results) ──────────┘   │
              └────────────────────────────────────────────────────┘
```

## 入口与启动流程

1. **用户执行** `oh` 或 `uv run oh`
2. **Typer CLI** (`cli.py:main()`) 解析命令行参数
3. **组装 RuntimeBundle**：
   - 加载 `Settings`（配置文件 + 环境变量 + CLI 参数）
   - 创建 `AnthropicApiClient`
   - 构建 `ToolRegistry`（注册 43 个工具）
   - 创建 `PermissionChecker`
   - 加载 `HookRegistry` + `SkillRegistry` + `Plugins`
   - 构建 System Prompt（环境 + skills + CLAUDE.md + Memory）
4. **创建 QueryEngine** 并进入交互循环
5. 用户每次输入调用 `query_engine.submit_message(prompt)` → `run_query()` 核心循环
