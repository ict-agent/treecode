# 权限系统

权限系统为工具执行提供安全边界——三种权限模式 + 路径规则 + 命令黑名单，确保 Agent 不会执行危险操作。

> 对应源码：`src/openharness/permissions/`

---

## 模块组成

| 文件 | 职责 |
|------|------|
| `checker.py` | `PermissionChecker` — 核心评估逻辑 |
| `modes.py` | `PermissionMode` 枚举 — 三种权限模式 |

## 三种权限模式

```python
class PermissionMode(str, Enum):
    DEFAULT = "default"       # 写操作需用户确认
    FULL_AUTO = "full_auto"   # 全部自动允许
    PLAN = "plan"             # 阻止所有写操作
```

| 模式 | 读操作 | 写操作 | 适用场景 |
|------|--------|--------|----------|
| **Default** | ✅ 自动允许 | ⚠️ 需要确认 | 日常开发 |
| **Full Auto** | ✅ 自动允许 | ✅ 自动允许 | 沙箱环境 |
| **Plan** | ✅ 自动允许 | ❌ 拒绝 | 大型重构前的审查规划 |

### 评估流程 `evaluate()`

```python
def evaluate(self, tool_name, *, is_read_only, file_path=None, command=None):
```

评估遵循严格的优先级顺序：

```
evaluate(tool_name, is_read_only, file_path, command)
│
├── 1. 显式工具拒绝列表
│     if tool_name in denied_tools → 拒绝
│
├── 2. 显式工具允许列表
│     if tool_name in allowed_tools → 允许
│
├── 3. 路径级规则 (glob 匹配)
│     for rule in path_rules:
│       if fnmatch(file_path, rule.pattern) and not rule.allow → 拒绝
│
├── 4. 命令拒绝模式
│     for pattern in denied_commands:
│       if fnmatch(command, pattern) → 拒绝
│
├── 5. Full Auto 模式
│     if mode == FULL_AUTO → 允许
│
├── 6. 只读工具
│     if is_read_only → 允许
│
├── 7. Plan 模式
│     if mode == PLAN → 拒绝 ("Plan mode blocks mutating tools")
│
└── 8. Default 模式
      → 拒绝 + requires_confirmation=True (需要用户确认)
```

### PermissionDecision 结果

```python
@dataclass(frozen=True)
class PermissionDecision:
    allowed: bool                    # 是否允许
    requires_confirmation: bool      # 是否需要用户确认
    reason: str                      # 决策原因（UI 展示用）
```

三种可能的结果：
1. `allowed=True` — 直接执行
2. `allowed=False, requires_confirmation=True` — 弹出确认对话框
3. `allowed=False, requires_confirmation=False` — 直接拒绝

## 路径规则

通过 `settings.json` 配置：

```json
{
  "permission": {
    "path_rules": [
      {"pattern": "/etc/*", "allow": false},
      {"pattern": "~/.ssh/*", "allow": false},
      {"pattern": "/tmp/*", "allow": true}
    ]
  }
}
```

使用 Python `fnmatch` 进行 glob 模式匹配：

```python
for rule in self._path_rules:
    if fnmatch.fnmatch(file_path, rule.pattern):
        if not rule.allow:
            return PermissionDecision(
                allowed=False,
                reason=f"Path {file_path} matches deny rule: {rule.pattern}",
            )
```

---

## 命令黑名单

```json
{
  "permission": {
    "denied_commands": ["rm -rf /", "DROP TABLE *", "sudo *"]
  }
}
```

工具传入的 `command` 参数会与黑名单进行 `fnmatch` 匹配：

```python
if command:
    for pattern in self._settings.denied_commands:
        if fnmatch.fnmatch(command, pattern):
            return PermissionDecision(
                allowed=False,
                reason=f"Command matches deny pattern: {pattern}",
            )
```

## 与 Agent Loop 的集成

在 `engine/query.py:_execute_tool_call()` 中，每次工具执行前都会调用权限检查：

```python
# 从工具输入中提取文件路径和命令
_file_path = str(tool_input.get("file_path", "")) or None
_command = str(tool_input.get("command", "")) or None

decision = context.permission_checker.evaluate(
    tool_name,
    is_read_only=tool.is_read_only(parsed_input),
    file_path=_file_path,
    command=_command,
)

if not decision.allowed:
    if decision.requires_confirmation and context.permission_prompt:
        # Default 模式：弹出确认对话框
        confirmed = await context.permission_prompt(tool_name, decision.reason)
        if not confirmed:
            return ToolResultBlock(..., is_error=True)
    else:
        # Plan 模式或规则拒绝：直接返回错误
        return ToolResultBlock(..., content=decision.reason, is_error=True)
```

## 模式切换

用户可以通过以下方式切换权限模式：

1. **CLI 参数**：`oh --permission-mode full_auto`
2. **命令**：`/permissions` → 从列表中选择
3. **工具**：`enter_plan_mode` / `exit_plan_mode`
4. **跳过权限**：`oh --dangerously-skip-permissions`（等同于 Full Auto）

### is_read_only 判断

每个工具可以覆盖 `is_read_only()` 方法来声明自己是否是只读操作。例如：

- `read_file` → 只读 (True)
- `bash` → 默认非只读 (False)
- `edit_file` → 非只读 (False)

只读工具在 Default 和 Full Auto 模式下都自动允许，不需要用户确认。
