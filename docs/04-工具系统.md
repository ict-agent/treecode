# 工具系统

工具系统是 Agent 与外部世界交互的接口——43 个工具覆盖文件操作、Shell 执行、Web 搜索、MCP、Agent 生成等能力。

> 对应源码：`src/openharness/tools/`

---

## 核心抽象

> 源码：[`tools/base.py`](../src/openharness/tools/base.py)

### BaseTool 基类

```python
class BaseTool(ABC):
    """Base class for all OpenHarness tools."""

    name: str                        # 工具名称（如 "bash", "read_file"）
    description: str                 # 工具描述（LLM 可见）
    input_model: type[BaseModel]     # Pydantic 输入模型

    @abstractmethod
    async def execute(self, arguments: BaseModel, context: ToolExecutionContext) -> ToolResult:
        """Execute the tool."""

    def is_read_only(self, arguments: BaseModel) -> bool:
        """是否只读操作（影响权限判断）。默认 False。"""
        return False

    def to_api_schema(self) -> dict[str, Any]:
        """生成 Anthropic API 格式的工具 Schema。"""
        return {
            "name": self.name,
            "description": self.description,
            "input_schema": self.input_model.model_json_schema(),
        }
```

每个工具需要实现：
1. **`name`** — 工具名，LLM 通过这个名字调用工具
2. **`description`** — 描述，帮助 LLM 理解何时使用该工具
3. **`input_model`** — Pydantic BaseModel 子类，定义输入参数
4. **`execute()`** — 异步执行方法

### ToolResult 结果

```python
@dataclass(frozen=True)
class ToolResult:
    output: str                         # 输出文本
    is_error: bool = False              # 是否执行错误
    metadata: dict[str, Any] = field()  # 额外元数据（如 returncode）
```

### ToolExecutionContext 上下文

```python
@dataclass
class ToolExecutionContext:
    cwd: Path                           # 当前工作目录
    metadata: dict[str, Any] = field()  # 共享元数据（含 tool_registry, ask_user_prompt 等）
```

### ToolRegistry 注册表

```python
class ToolRegistry:
    def register(self, tool: BaseTool) -> None
    def get(self, name: str) -> BaseTool | None
    def list_tools(self) -> list[BaseTool]
    def to_api_schema(self) -> list[dict[str, Any]]  # 生成所有工具的 API Schema
```

## JSON Schema 自动生成

工具的输入参数通过 Pydantic `model_json_schema()` 自动生成 JSON Schema，LLM 据此理解每个参数的类型和含义：

```python
class BashToolInput(BaseModel):
    command: str = Field(description="Shell command to execute")
    cwd: str | None = Field(default=None, description="Working directory override")
    timeout_seconds: int = Field(default=120, ge=1, le=600)
```

生成的 Schema 类似：
```json
{
  "name": "bash",
  "description": "Run a shell command in the local repository.",
  "input_schema": {
    "type": "object",
    "properties": {
      "command": {"type": "string", "description": "Shell command to execute"},
      "cwd": {"type": "string", "description": "Working directory override"},
      "timeout_seconds": {"type": "integer", "default": 120, "minimum": 1, "maximum": 600}
    },
    "required": ["command"]
  }
}
```

## 代表性工具实现详解

### BashTool — Shell 执行

> 源码：[`tools/bash_tool.py`](../src/openharness/tools/bash_tool.py)

```python
class BashTool(BaseTool):
    name = "bash"
    description = "Run a shell command in the local repository."
    input_model = BashToolInput

    async def execute(self, arguments, context):
        cwd = Path(arguments.cwd).expanduser() if arguments.cwd else context.cwd

        # 使用 asyncio.create_subprocess_exec 异步执行
        process = await asyncio.create_subprocess_exec(
            "/bin/bash", "-lc", arguments.command,
            cwd=str(cwd),
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
        )

        try:
            stdout, stderr = await asyncio.wait_for(
                process.communicate(),
                timeout=arguments.timeout_seconds,  # 默认 120s
            )
        except asyncio.TimeoutError:
            process.kill()
            return ToolResult(output="Command timed out...", is_error=True)

        # 合并 stdout + stderr，截断到 12000 字符
        text = "\n".join(parts).strip()
        if len(text) > 12000:
            text = f"{text[:12000]}\n...[truncated]..."

        return ToolResult(
            output=text,
            is_error=process.returncode != 0,
            metadata={"returncode": process.returncode},
        )
```

关键设计：
- 使用 `bash -lc` 以 login shell 执行（确保 PATH 等环境变量加载）
- `asyncio.wait_for` 实现超时控制
- 输出截断到 12,000 字符（避免超大输出撑爆 context window）
- `is_error` 由进程退出码决定

### FileEditTool — 文件编辑

> 源码：[`tools/file_edit_tool.py`](../src/openharness/tools/file_edit_tool.py)

```python
class FileEditTool(BaseTool):
    name = "edit_file"

    async def execute(self, arguments, context):
        path = _resolve_path(context.cwd, arguments.path)

        original = path.read_text(encoding="utf-8")
        if arguments.old_str not in original:
            return ToolResult(output="old_str was not found in the file", is_error=True)

        if arguments.replace_all:
            updated = original.replace(arguments.old_str, arguments.new_str)
        else:
            updated = original.replace(arguments.old_str, arguments.new_str, 1)

        path.write_text(updated, encoding="utf-8")
        return ToolResult(output=f"Updated {path}")
```

基于字符串替换的简洁编辑方式——LLM 提供 `old_str`（要替换的文本）和 `new_str`（替换后的文本）。

### AgentTool — Agent 生成

> 源码：[`tools/agent_tool.py`](../src/openharness/tools/agent_tool.py)

```python
class AgentTool(BaseTool):
    name = "agent"

    async def execute(self, arguments, context):
        task = await get_task_manager().create_agent_task(
            prompt=arguments.prompt,
            description=arguments.description,
            cwd=context.cwd,
            task_type=arguments.mode,  # local_agent / remote_agent / in_process_teammate
            model=arguments.model,
            api_key=os.environ.get("ANTHROPIC_API_KEY"),
            command=arguments.command,
        )

        if arguments.team:
            get_team_registry().add_agent(arguments.team, task.id)

        return ToolResult(output=f"Spawned {arguments.mode} task {task.id}")
```

与 `tasks/manager.py` 和 `coordinator/` 协同，实现多智能体生成。

## 工具与系统的集成点

每个工具在执行时经历完整的管道：

```
LLM 返回 tool_use → 
    ToolRegistry.get(name)         查找工具
    └→ input_model.model_validate() Pydantic 验证
    └→ is_read_only()              判断是否只读
    └→ PermissionChecker.evaluate() 权限检查
    └→ PreToolUse Hook             Hook 拦截
    └→ tool.execute()              执行
    └→ PostToolUse Hook            Hook 通知
→ 返回 ToolResultBlock
```

这个管道在 `engine/query.py:_execute_tool_call()` 中实现（详见 [02-Agent-Loop引擎](./02-Agent-Loop引擎.md)）。
