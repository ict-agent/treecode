# 记忆与上下文

记忆系统实现跨会话知识持久化，上下文系统将所有知识源（环境、技能、CLAUDE.md、Memory）组装成完整的 System Prompt。

> 对应源码：`src/treecode/memory/` + `src/treecode/prompts/`

## System Prompt 组装流程

> 源码：[`prompts/context.py`](../src/treecode/prompts/context.py)

```python
def build_runtime_system_prompt(settings, *, cwd, latest_user_prompt=None):
    sections = []

    # 1. 基础 prompt + 环境信息
    sections.append(build_system_prompt(custom_prompt=settings.system_prompt, cwd=str(cwd)))

    # 2. Fast Mode 提示（可选）
    if settings.fast_mode:
        sections.append("# Session Mode\nFast mode is enabled...")

    # 3. Output Style（可选，非 default 时注入）
    # 4. 推理设置
    sections.append(f"# Reasoning Settings\n- Effort: {settings.effort}\n- Passes: {settings.passes}")

    # 5. 可用技能列表
    skills_section = _build_skills_section(cwd)

    # 6. CLAUDE.md 项目指令
    claude_md = load_claude_md_prompt(cwd)

    # 7. Issue / PR Comments 上下文（截断到 12,000 字符）
    # 8. MEMORY.md 持久记忆
    # 9. 与当前输入相关的记忆检索

    return "\n\n".join(sections)
```

组装顺序（对应 `prompts/context.py` 的 append 顺序）：

```
┌─────────────────────────────┐
│  基础 System Prompt         │  ← 角色定义 + 环境信息
│  + 环境信息 (OS, CWD, Git)  │
├─────────────────────────────┤
│  Session Mode (可选)         │  ← Fast mode
├─────────────────────────────┤
│  Output Style (可选)         │  ← 非 default 时注入
├─────────────────────────────┤
│  Reasoning Settings          │  ← Effort + Passes
├─────────────────────────────┤
│  Available Skills            │  ← 可用技能列表
├─────────────────────────────┤
│  CLAUDE.md                   │  ← 项目级指令
├─────────────────────────────┤
│  Issue / PR Context          │  ← Issue 和 PR 评论
├─────────────────────────────┤
│  MEMORY.md                   │  ← 持久记忆入口
├─────────────────────────────┤
│  Relevant Memories           │  ← 与当前输入相关的记忆
└─────────────────────────────┘
```

```python
def _format_environment_section(env: EnvironmentInfo):
    lines = [
        "# Environment",
        f"- OS: {env.os_name} {env.os_version}",
        f"- Architecture: {env.platform_machine}",
        f"- Shell: {env.shell}",
        f"- Working directory: {env.cwd}",
        f"- Date: {env.date}",
        f"- Python: {env.python_version}",
    ]
    if env.is_git_repo:
        git_line = f"- Git: yes (branch: {env.git_branch})"
        lines.append(git_line)
    return "\n".join(lines)
```

## Memory 持久记忆

### 记忆路径（以代码实现为准）

> 源码：[`memory/paths.py`](../src/treecode/memory/paths.py)

```python
def get_project_memory_dir(cwd: str | Path) -> Path:
    """返回项目记忆目录，使用 SHA1 哈希确保唯一性。"""
    path = Path(cwd).resolve()
    digest = sha1(str(path).encode("utf-8")).hexdigest()[:12]
    return get_data_dir() / "memory" / f"{path.name}-{digest}"

def get_memory_entrypoint(cwd: str | Path) -> Path:
    """返回项目记忆入口文件。"""
    return get_project_memory_dir(cwd) / "MEMORY.md"
```

**实际路径**：`~/.treecode/data/memory/<project>-<hash>/MEMORY.md`

路径示例：`~/.treecode/data/memory/TreeCode-a1b2c3d4e5f6/MEMORY.md`

### 目录结构

```
~/.treecode/data/memory/<project>-<hash>/
├── MEMORY.md          # 入口文件
└── topics/
    ├── coding-style.md
    └── project-notes.md
```

> **注意**：`/init` 命令会在项目下创建 `.treecode/memory/MEMORY.md`，但 `get_memory_entrypoint()` 实际读取 `~/.treecode/data/memory/<project>-<hash>/MEMORY.md`。两者是不同的路径。

### MEMORY.md 加载

> 源码：[`memory/memdir.py`](../src/treecode/memory/memdir.py)

```python
def load_memory_prompt(cwd, *, max_entrypoint_lines=200):
    memory_dir = get_project_memory_dir(cwd)
    entrypoint = get_memory_entrypoint(cwd)
    lines = [
        "# Memory",
        f"- Persistent memory directory: {memory_dir}",
        "- Use this directory to store durable user or project context...",
    ]
    if entrypoint.exists():
        content_lines = entrypoint.read_text().splitlines()[:max_entrypoint_lines]
        if content_lines:
            lines.extend(["", "## MEMORY.md", "```md", *content_lines, "```"])
    return "\n".join(lines)
```

## 设计要点

1. **集中存储**：项目记忆统一存放在 `~/.treecode/data/memory/<project>-<hash>/`，使用 SHA1 哈希确保不同路径的同名项目互不冲突
2. **惰性加载**：记忆仅在构建 system prompt 时加载
3. **大小限制**：`max_entrypoint_lines` 防止 MEMORY.md 过大撑爆 context window
4. **相关性检索**：根据当前用户输入匹配最相关的记忆文件，而非全部加载
5. **12K 截断**：Issue/PR Context 内容截断到 12,000 字符
